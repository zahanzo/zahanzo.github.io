// Estrutura de nó, inserção/remoção, NULL como sentinela e proteção contra Buffer Overflow

Por que listas encadeadas?

Arrays têm tamanho fixo (stack) ou exigem realloc caro (heap). Uma lista encadeada resolve isso: cada elemento (nó) armazena o dado e um ponteiro para o próximo nó. O tamanho cresce e diminui dinamicamente, com custo O(1) para inserção/remoção no início ou quando você tem o ponteiro do nó anterior.

Estrutura de um nó

clinked_list.h
#include <stdlib.h>
#include <stdio.h>

/* A struct referencia a si mesma — o ponteiro precisa do tag */
typedef struct No {
    int        dado;
    struct No *proximo; /* aponta para o próximo nó, ou NULL */
} No;

/* Cria um novo nó no heap */
No* criar_no(int valor) {
    No *novo = (No*) malloc(sizeof(No));
    if (!novo) return NULL;
    novo->dado     = valor;
    novo->proximo  = NULL; /* sentinela: nenhum próximo */
    return novo;
}

NULL como sentinela — a convenção de segurança

NULL é o endereço zero — garantidamente não mapeado em qualquer processo. Usá-lo como marcador do final da lista é uma convenção universalmente válida e verificável. Toda travessia de lista deve checar NULL antes de desreferenciar.

head→[10|*]→[20|*]→[30|NULL]
nó 1 nó 2 nó 3 (final)

Operações fundamentais

clista_ops.c
/* Inserção no início — O(1) */
void inserir_inicio(No **head, int valor) {
    No *novo = criar_no(valor);
    if (!novo) return;
    novo->proximo = *head;  /* novo aponta para o antigo head */
    *head = novo;           /* head agora é o novo nó */
}

/* Inserção no final — O(n) */
void inserir_final(No **head, int valor) {
    No *novo = criar_no(valor);
    if (!novo) return;

    if (*head == NULL) {
        *head = novo;
        return;
    }
    No *atual = *head;
    while (atual->proximo != NULL) { /* anda até o último */
        atual = atual->proximo;
    }
    atual->proximo = novo;
}

/* Remoção por valor — O(n) */
void remover(No **head, int valor) {
    No *atual    = *head;
    No *anterior = NULL;

    while (atual != NULL && atual->dado != valor) {
        anterior = atual;
        atual    = atual->proximo;
    }
    if (atual == NULL) return; /* não encontrado */

    if (anterior == NULL) {
        *head = atual->proximo; /* remove o head */
    } else {
        anterior->proximo = atual->proximo; /* bypassa o nó */
    }
    free(atual);
}

/* Impressão com guard de NULL */
void imprimir(const No *head) {
    while (head != NULL) {
        printf("%d → ", head->dado);
        head = head->proximo;
    }
    puts("NULL");
}

/* Liberação total — OBRIGATÓRIO para evitar memory leak */
void liberar_lista(No **head) {
    No *atual = *head;
    while (atual != NULL) {
        No *prox = atual->proximo; /* salva antes de free! */
        free(atual);
        atual = prox;
    }
    *head = NULL;
}

Programa completo

cmain_lista.c
int main(void) {
    No *lista = NULL; /* lista vazia — head começa em NULL */

    inserir_inicio(&lista, 10);
    inserir_inicio(&lista, 5);
    inserir_final(&lista,  20);
    inserir_final(&lista,  30);

    imprimir(lista); /* 5 → 10 → 20 → 30 → NULL */

    remover(&lista, 10);
    imprimir(lista); /* 5 → 20 → 30 → NULL */

    liberar_lista(&lista);
    printf("head após free: %p\n", (void*)lista); /* (nil) */
    return 0;
}

Como listas encadeadas protegem contra Buffer Overflow

Arrays de tamanho fixo são a principal fonte de buffer overflow: se a entrada excede o tamanho reservado, você sobrescreve memória adjacente. Com uma lista encadeada, cada elemento é alocado individualmente no heap com malloc — não há buffer de tamanho fixo a transbordar. A lista cresce exatamente o necessário.

// Tradeoff: Listas encadeadas têm custo de memória (ponteiro extra por nó) e pior localidade de cache (nós espalhados no heap). Para a maioria dos casos onde o tamanho é imprevisível e segurança importa, o tradeoff vale. Para performance pura com tamanho conhecido, arrays/vetores vencen.
