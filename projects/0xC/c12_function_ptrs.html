0x0C · Ponteiros de Função

// Declaração, despacho dinâmico, callbacks, plugins e bases de RTOS

Funções são código na memória

Quando o linker resolve os símbolos, cada função recebe um endereço fixo na seção .text do binário. Um ponteiro de função é uma variável que armazena esse endereço. Assim como um int* aponta para um inteiro, um ponteiro de função aponta para código executável e pode ser chamado.

Isso habilita despacho dinâmico: a decisão de qual função chamar é tomada em runtime, não em compile-time. É a base de callbacks, sistemas de plugins, tabelas de dispatch (vtables de C++), e handlers de interrupção em RTOS.

Sintaxe de ponteiro de função

cfn_ptr_basico.c
#include <stdio.h>

int somar(int a, int b)     { return a + b; }
int subtrair(int a, int b) { return a - b; }
int multiplicar(int a, int b){ return a * b; }

int main(void) {
    /* Declara ponteiro de função:
       int (*nome)(int, int) — aponta para função que recebe 2 ints e retorna int */
    int (*op)(int, int);

    op = somar;                   /* sem & — nome da função já é endereço */
    printf("10 + 3 = %d\n", op(10, 3)); /* 13 */

    op = subtrair;
    printf("10 - 3 = %d\n", op(10, 3)); /* 7  */

    op = multiplicar;
    printf("10 * 3 = %d\n", op(10, 3)); /* 30 */

    printf("Endereço de multiplicar: %p\n", (void*)(op));

    return 0;
}

typedef para legibilidade

A sintaxe de ponteiro de função aninhada fica ilegível rapidamente. Use typedef para criar um alias de tipo limpo:

cfn_ptr_typedef.c
/* Define o tipo Operacao como ponteiro para função (int, int) → int */
typedef int (*Operacao)(int, int);

void executar(Operacao fn, int a, int b) {
    printf("Resultado: %d\n", fn(a, b));
}

int main(void) {
    executar(somar,      5, 3); /* 8  */
    executar(subtrair,   5, 3); /* 2  */
    executar(multiplicar,5, 3); /* 15 */
    return 0;
}

Tabela de dispatch — motor de plugins

A técnica mais poderosa: um array de ponteiros de função. Cada índice corresponde a um comando/opcode. A seleção do handler é O(1) — um único acesso ao array — em vez de uma cadeia de if/else ou switch.

cdispatch_table.c
#include <stdio.h>
#include <string.h>

typedef void (*Handler)(const char *args);

void cmd_status (const char *a) { printf("[STATUS] sistema online. args=%s\n", a); }
void cmd_reset  (const char *a) { printf("[RESET]  reiniciando... args=%s\n", a); }
void cmd_debug  (const char *a) { printf("[DEBUG]  dump de memória. args=%s\n", a); }

typedef struct {
    const char *nome;
    Handler     fn;
} Comando;

int main(void) {
    Comando tabela[] = {
        { "status", cmd_status },
        { "reset",  cmd_reset  },
        { "debug",  cmd_debug  },
        { NULL, NULL }             /* sentinela de fim de tabela */
    };

    const char *entrada = "reset";

    for (int i = 0; tabela[i].nome != NULL; i++) {
        if (strcmp(tabela[i].nome, entrada) == 0) {
            tabela[i].fn("arg1");  /* despacho dinâmico */
            return 0;
        }
    }
    fprintf(stderr, "Comando desconhecido: %s\n", entrada);
    return 1;
}

Callbacks — qsort como exemplo canônico

A função qsort da stdlib é o exemplo mais famoso de callback em C. Ela implementa quicksort genérico e recebe um ponteiro de função de comparação que define a ordenação:

cqsort_callback.c
#include <stdlib.h>
#include <stdio.h>

/* Comparador: recebe void* genéricos, retorna negativo/0/positivo */
int cmp_crescente(const void *a, const void *b) {
    return *(const int*)a - *(const int*)b;
}

int cmp_decrescente(const void *a, const void *b) {
    return *(const int*)b - *(const int*)a;
}

int main(void) {
    int arr[] = {5, 2, 8, 1, 9, 3};
    int n = sizeof(arr) / sizeof(arr[0]);

    qsort(arr, n, sizeof(int), cmp_crescente);
    /* arr = {1, 2, 3, 5, 8, 9} */

    qsort(arr, n, sizeof(int), cmp_decrescente);
    /* arr = {9, 8, 5, 3, 2, 1} */
    return 0;
}

Base para RTOS — tabela de tarefas

Em sistemas embarcados de tempo real (RTOS), o scheduler mantém uma tabela de tarefas onde cada entrada é um ponteiro de função. O scheduler chama cada tarefa sequencialmente (cooperative) ou por prioridade/timer (preemptive). O padrão é exatamente a tabela de dispatch vista acima — com metadados de prioridade e período:

crtos_scheduler_base.c
typedef void (*Tarefa)(void);

typedef struct {
    Tarefa    fn;
    int       periodo_ms;
    int       ultimo_run_ms;
} EntradaTarefa;

void tarefa_sensor  (void) { /* lê ADC */ }
void tarefa_controle(void) { /* PID loop */ }
void tarefa_comms   (void) { /* envia UART */ }

EntradaTarefa scheduler[] = {
    { tarefa_sensor,   10,  0 }, /* a cada 10ms */
    { tarefa_controle, 20,  0 }, /* a cada 20ms */
    { tarefa_comms,    100, 0 }, /* a cada 100ms */
};

void run_scheduler(int agora_ms) {
    int n = sizeof(scheduler) / sizeof(scheduler[0]);
    for (int i = 0; i < n; i++) {
        if (agora_ms - scheduler[i].ultimo_run_ms >= scheduler[i].periodo_ms) {
            scheduler[i].fn();              /* despacho dinâmico */
            scheduler[i].ultimo_run_ms = agora_ms;
        }
    }
}

⚠ Segurança — Function Pointer Hijacking: Em exploits de corrupção de memória, um atacante que consegue sobrescrever um ponteiro de função (em uma struct no heap, por exemplo) pode redirecionar a execução para código arbitrário — um hijack de fluxo de controle. Mitigações: CFI (Control Flow Integrity) — compiladores modernos com -fsanitize=cfi inserem verificações que garantem que o destino de um call indireto é uma função válida do tipo esperado.
