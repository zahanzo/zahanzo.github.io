0x08 · Ponteiros e Segmentation Fault

// Declaração, desreferenciamento, aritmética de ponteiros e SegFault

O que é um ponteiro?

Um ponteiro é uma variável que armazena um endereço de memória. Enquanto int x = 42 armazena o valor 42, int *p = &x armazena o endereço onde 42 vive. O tamanho de um ponteiro é sempre o mesmo, independente do tipo apontado: 8 bytes em sistemas 64-bit.

cponteiros_basico.c
#include <stdio.h>

int main(void) {
    int  x  = 42;
    int *p  = &x;   /* p aponta para x */

    printf("x = %d\n",        x);        /* 42             */
    printf("&x = %p\n",       &x);       /* 0x7fff...      */
    printf("p = %p\n",        p);        /* mesmo que &x   */
    printf("*p = %d\n",       *p);       /* 42 (desrefer.) */
    printf("sizeof(p) = %zu\n", sizeof(p)); /* 8 (64-bit)     */

    *p = 100; /* modifica x através de p */
    printf("x agora = %d\n", x); /* 100 */

    return 0;
}

A sintaxe de ponteiros — lida da direita para a esquerda

int *p← "p é um ponteiro para int"
int **pp← "pp é um ponteiro para ponteiro para int"
int *arr[5]← "arr é um array de 5 ponteiros para int"
int (*fn)(void)← "fn é um ponteiro para função que retorna int"
const int *p← "p aponta para int constante (não pode mudar *p)"
int * const p← "p é ponteiro constante (não pode mudar p em si)"

Aritmética de ponteiros

Ponteiros suportam aritmética. Mas a unidade não é bytes — é sizeof(tipo). Quando você faz p + 1 em um int *p, avança 4 bytes (sizeof int), não 1. Isso é o que torna ponteiros e arrays intercambiáveis em C.

caritmetica_ponteiros.c
int arr[] = {10, 20, 30, 40};
int *p    = arr; /* arr decai para &arr[0] */

printf("%d\n", *p);       /* 10 — arr[0] */
printf("%d\n", *(p + 1)); /* 20 — arr[1] (avança 4 bytes) */
printf("%d\n", *(p + 2)); /* 30 — arr[2] (avança 8 bytes) */

/* Equivalente com notação de array: */
printf("%d\n", p[3]);     /* 40 — mesmo que *(p+3) */

/* Subtração de ponteiros — retorna ptrdiff_t (número de elementos) */
int *inicio = &arr[0];
int *fim    = &arr[3];
printf("distância: %td\n", fim - inicio); /* 3 (elementos, não bytes) */

Segmentation Fault — lendo a mensagem do SO

Um Segmentation Fault (SIGSEGV) ocorre quando o processo tenta acessar uma região de memória que não tem permissão para acessar. O hardware (MMU — Memory Management Unit) detecta a violação e o kernel envia o sinal SIGSEGV, que por padrão termina o processo.

Causas comuns:

csegfault_causas.c
/* 1. Ponteiro NULL desreferenciado */
int *p = NULL;
*p = 42;  /* SIGSEGV — endereço 0x0 não é mapeado */

/* 2. Ponteiro não inicializado (lixo de memória como endereço) */
int *q;   /* valor indeterminado */
*q = 1;   /* SIGSEGV — endereço aleatório */

/* 3. Array out-of-bounds */
int arr[5];
arr[10] = 99; /* Undefined behavior — pode ou não ser SegFault */

/* 4. Use-after-free (próximo capítulo) */
int *r = malloc(sizeof(int));
free(r);
*r = 5;   /* Undefined behavior — heap já não pertence ao processo */

⚠ Out-of-bounds sem SegFault = pior cenário: Um acesso fora dos limites de um array pode não gerar SegFault se o endereço acidentalmente cair em outra região válida do processo. Nesse caso, você corrompe silenciosamente dados adjacentes — o bug pode se manifestar muito depois, em uma parte completamente diferente do código. Use AddressSanitizer: compile com -fsanitize=address para detectar isso.

Ponteiros para void — o tipo genérico

void * é um ponteiro sem tipo. Ele pode armazenar qualquer endereço, mas não pode ser desreferenciado diretamente — você precisa fazer cast primeiro. É usado em funções genéricas como malloc(), memcpy() e em callbacks.

cvoid_ptr.c
void imprimir_bytes(const void *ptr, size_t n) {
    const unsigned char *bytes = (const unsigned char*)ptr;
    for (size_t i = 0; i < n; i++) {
        printf("%02X ", bytes[i]);
    }
    putchar('\n');
}

int main(void) {
    double pi = 3.14159265358979;
    imprimir_bytes(&pi, sizeof(pi));
    /* Imprime: 6E 86 1B F0 F9 21 09 40 (IEEE 754, little-endian) */
    return 0;
}
