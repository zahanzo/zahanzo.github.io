O que é a Stack?

A Stack (pilha) é uma região de memória gerenciada automaticamente pelo compilador e pela CPU. Ela segue a disciplina LIFO — Last In, First Out: o último bloco alocado é o primeiro a ser liberado. Essa propriedade é perfeita para modelar chamadas de função aninhadas.

Em x86-64, a stack cresce para endereços mais baixos. Dois registradores a governam: RSP (Stack Pointer) aponta para o topo atual da stack, e RBP (Base Pointer) aponta para a base do frame atual.

Stack Frames

Cada chamada de função cria um stack frame — um bloco de memória reservado na stack para as variáveis locais, o endereço de retorno e os registradores salvos daquela invocação. Quando a função retorna, o frame é "desempilhado" movendo RSP de volta.

cstack_frames.c
#include <stdio.h>

int soma(int a, int b) {
    int resultado = a + b; /* variável local no frame de soma() */
    return resultado;
}

int main(void) {
    int x = 10;
    int y = 20;
    int s = soma(x, y); /* cria frame de soma() */
    printf("Soma: %d\n", s);
    return 0;
}

O que acontece na memória (Assembly implícito)

Stack (alto→baixo)
─────────────────
... (main frame)
[rbp-8]0x0000000A← x = 10
[rbp-16]0x00000014← y = 20
[rbp-20]0x0000001E← s = 30 (após retorno)
─────────────────
(soma frame)
[ret addr]0x00401185← endereço de retorno em main
[saved rbp]0x7ffe1234← rbp de main salvo
[rbp-4]0x0000001E← resultado = 30
RSP ────────────← topo atual

Ciclo de vida de variáveis locais

Variáveis locais existem apenas durante a execução da função. Quando a função retorna e o RSP é restaurado, aquele espaço de memória não é zerado — o compilador simplesmente ignora aquela região. Os bytes antigos permanecem fisicamente na RAM até serem sobrescritos por um novo frame.

Isso é chamado de lixo de memória (uninitialized memory / garbage value). Se você lê uma variável local sem inicializá-la, lê exatamente esse lixo.

cgarbage_value.c
#include <stdio.h>

void preenche(void) {
    int segredo = 0xDEADBEEF; /* dado "sensível" no stack frame */
    printf("segredo está em: %p\n", (void*)&segredo);
}

void vaza(void) {
    int lixo; /* NÃO inicializada — ocupa o mesmo espaço do frame anterior */
    printf("lixo lido: 0x%X\n", lixo); /* pode imprimir 0xDEADBEEF! */
}

int main(void) {
    preenche();
    vaza();    /* reutiliza o mesmo espaço de stack */
    return 0;
}

⚠ Information Leak — CVE real: Essa técnica é a base de ataques de information disclosure. Se uma função copia dados sensíveis (chave criptográfica, senha) para a stack e não os zera antes de retornar, um atacante que consiga ler memória naquela região (via bug de leitura fora dos limites) pode extrair esses dados. A mitigação é: sempre zere buffers sensíveis com memset(buffer, 0, sizeof(buffer)) antes de retornar — e use memset_explicit() ou SecureZeroMemory() para evitar que o compilador otimize o memset para fora.

Stack Overflow (o real)

A stack tem tamanho fixo (tipicamente 8 MB no Linux). Recursão infinita ou alocação de arrays gigantes na stack (ex: char buf[10000000];) consome toda a stack até o sistema operacional levantar um SIGSEGV — o famoso segmentation fault por stack overflow. Monitore o uso com ulimit -s.
