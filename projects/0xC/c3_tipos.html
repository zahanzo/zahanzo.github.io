0x03 · Tipos Primitivos e Bits

// Mapeamento de bits, modificadores signed/unsigned e representação na memória

Tipos não são abstrações — são garantias de tamanho

Em C, um tipo primitivo define dois contratos com o compilador: quantos bytes de memória a variável ocupa e como interpretar o padrão de bits armazenado (signed, unsigned, float). Tudo o mais é consequência.

csizeof_demo.c
#include <stdio.h>

int main(void) {
    printf("char    : %zu byte(s)\n",  sizeof(char));
    printf("short   : %zu byte(s)\n",  sizeof(short));
    printf("int     : %zu byte(s)\n",  sizeof(int));
    printf("long    : %zu byte(s)\n",  sizeof(long));
    printf("float   : %zu byte(s)\n",  sizeof(float));
    printf("double  : %zu byte(s)\n",  sizeof(double));
    printf("pointer : %zu byte(s)\n",  sizeof(void*));
    return 0;
}

⚠ Atenção: O padrão C não garante tamanhos fixos para int e long — eles dependem da ABI da plataforma. Para código portável e preciso, use <stdint.h>: int8_t, uint32_t, int64_t.

Signed vs Unsigned — Complemento de dois

Um int de 4 bytes armazena 32 bits. A diferença entre signed int e unsigned int é somente a interpretação do bit mais significativo (MSB — Most Significant Bit). O padrão de bits na memória é idêntico — o que muda é como a CPU interpreta as operações aritméticas.

Em complemento de dois (representação padrão para signed), o MSB tem peso negativo. Para um byte (char):

Bit 7 (MSB)peso: -128← sinal
Bit 6peso: +64
Bit 5peso: +32
Bit 4peso: +16
Bit 3peso: +8
Bit 2peso: +4
Bit 1peso: +2
Bit 0 (LSB)peso: +1

O padrão 11111111 (0xFF) interpretado como unsigned char = 255. Interpretado como signed char = -1 (−128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = −1).

csigned_unsigned.c
#include <stdio.h>

int main(void) {
    unsigned char u = 255;
    signed char   s = (signed char)u; /* reinterpreta os mesmos bits */

    printf("unsigned: %u\n", u);  /* 255  */
    printf("signed  : %d\n", s);  /* -1   */
    printf("hex     : 0x%X\n", u); /* 0xFF */
    return 0;
}

Integer Overflow — o bug silencioso

Quando um valor ultrapassa o limite do tipo, ocorre overflow. Para tipos unsigned, o comportamento é definido pelo padrão C: o valor "enrola" (wrap-around) no módulo 2ⁿ. Para tipos signed, o comportamento é undefined behavior — o compilador pode fazer qualquer coisa.

coverflow.c
unsigned char x = 255;
x++;           /* 255 + 1 = 256 % 256 = 0 (wrap-around definido) */
printf("%u\n", x); /* Output: 0 */

signed int    y = 2147483647; /* INT_MAX */
y++;           /* Undefined Behavior em signed! Não faça isso. */

⚠ CVE real: Integer overflows em cálculos de tamanho de buffer são a raiz de muitos heap overflows. Exemplo: size_t total = n_items * item_size; — se n_items é controlado pelo usuário e o resultado "enrola", malloc(total) aloca um buffer pequeno, mas o código escreve n_items * item_size bytes — overflow garantido. Sempre use __builtin_mul_overflow() ou cheque manualmente antes de multiplicar.

Tipos fixos com stdint.h

cstdint_demo.c
#include <stdint.h>
#include <stdio.h>

int main(void) {
    uint8_t  byte_val  = 0xFF;     /* Sempre 8 bits unsigned */
    int32_t  word_val  = -42;     /* Sempre 32 bits signed  */
    uint64_t qword_val = 0xDEADBEEFCAFEBABE;
    printf("0x%llX\n", (unsigned long long)qword_val);
    return 0;
}
