0x06 · scanf e Buffer Sanitization

// stdin, buffer do teclado, higienização e prevenção de loops infinitos

O que é stdin?

stdin é um stream de I/O — um canal de bytes entre o processo e a entrada padrão (geralmente o teclado). Ele é bufferizado por linha pelo SO: quando você pressiona Enter, os bytes digitados (incluindo o '\n') são enviados ao buffer do processo de uma vez.

A libc mantém um buffer interno para stdin. Funções como scanf, fgets e getchar leem desse buffer. O problema: quando scanf lê um inteiro e o usuário digita "42\n", o scanf consome 42 mas o '\n' permanece no buffer — aguardando a próxima leitura.

cscanf_basico.c
#include <stdio.h>

int main(void) {
    int numero;

    printf("Digite um inteiro: ");
    int ret = scanf("%d", &numero); /* & = endereço de numero */

    if (ret != 1) {
        fprintf(stderr, "[ERRO] Entrada inválida.\n");
        return 1;
    }
    printf("Você digitou: %d\n", numero);
    return 0;
}

O valor de retorno de scanf — NUNCA ignore

scanf retorna o número de itens convertidos com sucesso. Se o usuário digitar "abc" quando se espera um inteiro, scanf retorna 0 (zero conversões). Se o stream chega ao final (EOF), retorna EOF (-1). Ignorar esse retorno é um bug de segurança.

O loop infinito — o bug mais comum com scanf

Quando scanf falha em converter (ex: lê texto onde espera número), ele não consome os bytes inválidos. Eles ficam no buffer. Na próxima iteração do loop, scanf tenta novamente — e falha novamente — infinitamente.

cloop_infinito_bug.c — CÓDIGO QUEBRADO
int numero;
while (1) {
    printf("Digite um número: ");
    scanf("%d", &numero);   /* BUG: se digitar "abc", loop infinito! */
    printf("Recebido: %d\n", numero);
}

Higienização correta do buffer stdin

A solução é limpar o buffer quando a entrada é inválida. A idioma padrão para isso é consumir todos os caracteres restantes no buffer até encontrar um newline ou EOF:

cscanf_seguro.c
#include <stdio.h>

/* Limpa todos os bytes pendentes no buffer de stdin */
void limpar_buffer(void) {
    int c;
    while ((c = getchar()) != '\n' && c != EOF);
}

int main(void) {
    int numero;
    int valido = 0;

    while (!valido) {
        printf("Digite um inteiro positivo: ");

        if (scanf("%d", &numero) != 1) {
            fprintf(stderr, "[ERRO] Entrada não é um número.\n");
            limpar_buffer(); /* descarta os bytes inválidos */
        } else if (numero <= 0) {
            fprintf(stderr, "[ERRO] Número deve ser positivo.\n");
            limpar_buffer(); /* descarta o '\n' restante */
        } else {
            limpar_buffer(); /* boa prática: sempre limpa após leitura */
            valido = 1;
        }
    }

    printf("Número aceito: %d\n", numero);
    return 0;
}

fgets — a alternativa mais segura

Para leitura de strings, nunca use scanf("%s", buf) sem limite de tamanho — ele não tem limite e é vulnerável a buffer overflow. Use fgets: ela lê no máximo n-1 bytes e garante terminação com '\0'.

cfgets_seguro.c
#include <stdio.h>
#include <string.h>

#define MAX_NOME 64

int main(void) {
    char nome[MAX_NOME];

    printf("Nome: ");
    if (fgets(nome, sizeof(nome), stdin) == NULL) {
        fprintf(stderr, "Erro de leitura.\n");
        return 1;
    }

    /* fgets preserva o '\n' — remove-o */
    nome[strcspn(nome, "\n")] = '\0';

    printf("Olá, %s!\n", nome);
    return 0;
}

⚠ gets() é proibido: A função gets() foi removida do padrão C11. Ela não tem limite de tamanho e é garantia de buffer overflow. Qualquer código que a usa é vulnerável. Use sempre fgets(buf, sizeof(buf), stdin).
