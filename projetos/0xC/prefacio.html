Por que Baixo Nível ainda importa?

Em 2026, a maioria dos desenvolvedores programa com abstrações sobre abstrações: frameworks que escondem o garbage collector que esconde o heap que esconde o sistema de memória virtual que esconde os registradores da CPU. Isso não é um problema — até ser.

Bugs de segurança críticos — buffer overflows, use-after-free, double-free, integer overflows — continuam sendo as principais classes de vulnerabilidade exploradas em produção. CVEs com CVSS 9.8 em kernels, navegadores e bibliotecas de rede. Todos têm a mesma raiz: o programador não sabia exatamente o que o hardware estava fazendo.

O que este laboratório cobre

Este não é um curso de sintaxe. Cada módulo explica o que acontece fisicamente: como a CPU alocou o stack frame, qual instrução Assembly foi emitida, como o linker resolveu os símbolos. C é a lupa que torna esse mecanismo visível.

A grade curricular vai do zero absoluto — a estrutura mínima de um binário ELF — até ponteiros de função e a base para construção de sistemas embarcados (RTOS). Cada capítulo termina com um vetor de ataque real derivado do conteúdo, para que a teoria e a segurança caminhem juntas.

Como estudar

Não leia passivamente. Cada bloco de código foi escrito para ser compilado e inspecionado. Use gcc -O0 -g e examine com gdb ou objdump -d. Quando possível, rode em Linux nativo ou WSL2 — o comportamento de memória é determinístico e auditável.

// Convenção de Código — Todos os exemplos compilam com: gcc -Wall -Wextra -O0 -g -o output file.c. Warnings são features, não ruído.

A filosofia deste lab

Dominar C não é sobre usar C em produção. É sobre ter um modelo mental preciso de como a memória funciona. Com esse modelo, você escreve Python mais seguro, audita Rust com mais confiança e lê um CVE sem precisar do artigo de explicação.
