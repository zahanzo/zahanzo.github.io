0x01 · Estrutura do Binário e main()

// ELF, Entry Points, Stack Frames e Exit Codes

O programa mínimo

Antes de qualquer Hello World, precisamos entender o que um programa é do ponto de vista do sistema operacional. Um binário compilado em Linux é um arquivo no formato ELF (Executable and Linkable Format). Ele contém seções: .text (código), .data (variáveis globais inicializadas), .bss (globais não-inicializadas), e metadados que o kernel usa para carregar o processo na memória.

cestrutura_minima.c
#include <stdio.h>

int main(void) {
    printf("0xC_ online.\n");
    return 0;
}

O que é a função main()?

Do ponto de vista do programador, main() é onde o programa começa. Do ponto de vista do sistema operacional, ela não é o verdadeiro ponto de entrada. O ELF define um símbolo chamado _start, que é código gerado pelo linker (da libc/crt0) responsável por:

  1. Preparar os argumentos argc e argv a partir do stack do processo
  2. Chamar funções de inicialização da libc
  3. Chamar a função main()
  4. Passar o retorno de main() para exit()

Você pode ver isso em ação: objdump -d seu_binario | grep -A 20 "_start"

Assinaturas válidas de main()

cmain_signatures.c
/* Forma 1: sem argumentos */
int main(void) { return 0; }

/* Forma 2: com argumentos de linha de comando */
int main(int argc, char *argv[]) {
    /* argc = número de argumentos (pelo menos 1: o próprio programa) */
    /* argv = array de strings com cada argumento                      */
    printf("Programa: %s\n", argv[0]);
    return 0;
}

Exit Codes — A linguagem do processo

O valor retornado por main() é o Exit Code do processo. Ele é uma convenção entre o processo filho e o processo pai (que no terminal é o shell). O tipo é um int, mas o sistema operacional usa apenas os 8 bits menos significativos (exit_code & 0xFF), logo valores válidos são 0–255.

cexit_codes.c
#include <stdlib.h>  /* EXIT_SUCCESS, EXIT_FAILURE */

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "Uso: %s <arquivo>\n", argv[0]);
        return EXIT_FAILURE; /* EXIT_FAILURE == 1 */
    }
    /* Processamento normal... */
    return EXIT_SUCCESS; /* EXIT_SUCCESS == 0  */
}

No shell, inspecione o exit code do último processo com echo $?. Scripts de CI/CD usam isso para determinar se um build passou ou falhou.

Endereço Alto│[ ambiente ]← variáveis de ambiente
...│[ argv[] ]← strings dos argumentos
...│[ stack ]← cresce para baixo ↓
...│[ heap ]← cresce para cima ↑
...│[ .bss ]← globais não inicializadas (zeroed)
...│[ .data ]← globais inicializadas
Endereço Baixo│[ .text ]← código executável (somente leitura)

Seções ELF na prática

Compile qualquer exemplo e inspecione as seções com readelf -S binario ou size binario. O comando size mostra em bytes: text, data e bss — três números que resumem a anatomia do seu programa.

⚠ Vetor de ataque: A seção .text é marcada como somente leitura e executável. Se um overflow conseguir redirecionar o instruction pointer (RIP/EIP) para dados controláveis pelo atacante, o S.O. levanta uma exceção por violação de permissão — proteção chamada DEP/NX (No-Execute). Mas isso só funciona se o binário foi compilado com as flags corretas.
