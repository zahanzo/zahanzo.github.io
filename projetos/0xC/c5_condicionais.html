Como a CPU avalia condições

Em C, toda expressão condicional se reduz a um inteiro: zero é falso, qualquer valor não-zero é verdadeiro. Não existe tipo booleano nativo em C89/C90 — mas <stdbool.h> (C99+) adiciona bool, true e false como macros sobre int.

Quando a CPU executa uma instrução de comparação (cmp em x86), ela subtrai os operandos e armazena os resultados em bits especiais chamados FLAGS no registrador EFLAGS/RFLAGS: ZF (zero flag), SF (sign flag), CF (carry flag), OF (overflow flag). As instruções de salto condicional (je, jl, jg...) checam esses flags.

cif_else.c
#include <stdbool.h>
#include <stdio.h>

int main(void) {
    int temperatura = 38;

    if (temperatura > 37) {
        puts("[ALERTA] Temperatura acima do normal.");
    } else if (temperatura == 37) {
        puts("[OK] Temperatura normal.");
    } else {
        puts("[ALERTA] Hipotermia.");
    }

    /* bool do stdbool.h: macro sobre _Bool */
    bool alerta = (temperatura != 37);
    printf("alerta = %d\n", alerta); /* 1 (true) */
    return 0;
}

O Assembly gerado pelo if

asmx86-64 — if (temperatura > 37)
    movl    -4(%rbp), %eax     ; carrega temperatura para eax
    cmpl    $37, %eax          ; compara: eax - 37 (ajusta flags)
    jle     .L_else            ; se ≤, pula para else (ZF ou SF≠OF)
    ; corpo do if...
    jmp     .L_end
.L_else:
    ; corpo do else...
.L_end:

⚠ Erro clássico — = vs ==: if (x = 5) é atribuição (sempre verdadeiro, x vira 5). if (x == 5) é comparação. O compilador com -Wall avisa sobre isso. A técnica "Yoda condition" — if (5 == x) — elimina o risco: if (5 = x) seria erro de compilação.

Switch — tabela de saltos

O switch é mais eficiente que uma cadeia de if/else if quando os casos são inteiros densos. O compilador frequentemente gera uma jump table: um array de endereços onde o índice é o valor do switch. A seleção do caso vira uma única instrução de salto indireto — O(1) independente do número de casos.

cswitch_demo.c
int main(void) {
    int opcode = 0x02;

    switch (opcode) {
        case 0x00:
            puts("NOP");
            break; /* sem break → fallthrough para o próximo case! */
        case 0x01:
            puts("LOAD");
            break;
        case 0x02:
            puts("STORE");
            break;
        default:
            puts("OPCODE INVÁLIDO");
    }
    return 0;
}

Operadores Bitwise — manipulação direta de bits

Operadores bitwise operam bit a bit nos operandos. São fundamentais em programação de sistemas, drivers, protocolos e criptografia.

cbitwise.c
unsigned int flags = 0b00000000;

/* SET bit 3: usa OR com máscara */
flags |= (1 << 3);  /* flags = 0b00001000 */

/* CHECK bit 3: usa AND com máscara */
if (flags & (1 << 3)) { puts("bit 3 está setado"); }

/* CLEAR bit 3: usa AND com NOT da máscara */
flags &= ~(1 << 3); /* flags = 0b00000000 */

/* TOGGLE bit 3: usa XOR com máscara */
flags ^= (1 << 3);  /* alterna o bit */

/* Divisão/Multiplicação por potência de 2 via shift */
int x = 64;
x >>= 1; /* x = 32  (divide por 2) */
x <<= 2; /* x = 128 (multiplica por 4) */

// Aplicação real: Em drivers de hardware e registradores de periféricos, cada bit tem um significado. Por exemplo, no registrador de controle de uma UART: bit 0 = TX enable, bit 1 = RX enable, bit 2 = interrupt enable. Você usa exatamente essas técnicas bitwise para configurar o hardware sem afetar os outros bits.
